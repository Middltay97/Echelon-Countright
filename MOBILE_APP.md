# CountRight Android (Capacitor) — MDM deployment

This project ships **CountRight** to Chainway C6000 scanners as a native
Android app built with **Capacitor**. The native shell loads the published
Lovable web app remotely (`https://aagcountright.lovable.app/m`), so UI/logic
changes go live instantly via Lovable Publish — you only rebuild the APK when
you change the icon, name, permissions, or version.

Published URL: **https://aagcountright.lovable.app** · Mobile entry: `/m`

---

## ⚡ MDM deployment (ManageEngine) — GitHub Actions

Builds run in CI at `.github/workflows/android.yml`. Every push to `main`
(and every manual dispatch) produces a signed `countright-release.apk` +
`countright-release.aab` you can download from the Actions run page and
upload to ManageEngine.

### One-time setup

1. **Generate a signing keystore** on your laptop (needs JDK `keytool`):
   ```bash
   keytool -genkeypair -v \
     -keystore countright.keystore -alias countright \
     -keyalg RSA -keysize 2048 -validity 10000 \
     -storepass <PASSWORD> -keypass <PASSWORD> \
     -dname "CN=CountRight, O=AAG, C=US"
   ```
   **Back up `countright.keystore` + password.** Lose them and you can never
   ship an update to the same package — MDM treats a re-keyed APK as a
   different app.

2. **Add 4 repo secrets** in GitHub → Settings → Secrets and variables → Actions:

   | Secret | Value |
   | --- | --- |
   | `ANDROID_KEYSTORE_BASE64` | `base64 -w0 countright.keystore` |
   | `ANDROID_KEYSTORE_PASSWORD` | your `-storepass` |
   | `ANDROID_KEY_ALIAS` | `countright` |
   | `ANDROID_KEY_PASSWORD` | your `-keypass` |

3. **Trigger a build**: push to `main`, or open the workflow in Actions →
   `Run workflow`. Download the `countright-apk` artifact.

4. **Wire `assetlinks.json`** (optional but recommended for fullscreen).
   Grab the SHA-256 fingerprint from the workflow's
   *"Print SHA-256 cert fingerprint"* step and paste it into
   `src/routes/[.]well-known.assetlinks[.]json.ts` (replace both
   `REPLACE_WITH_…` placeholders), then **Publish** in Lovable.

### Each release

- Bump `versionCode` (and optionally `versionName`) in
  `android/app/build.gradle`, commit, push. CI produces a fresh APK.
  The package ID is unchanged so ManageEngine treats it as an upgrade.
- Or push a tag `android-vX.Y.Z` to also attach the APK + AAB to a
  GitHub Release.

### On a C6000 in ManageEngine

1. ManageEngine MDM → App Repository → **Add Enterprise App** → upload the APK.
2. Create a distribution policy targeting your C6000 device group;
   choose **Silent install** (works because devices are MDM-enrolled).
3. Optionally mark it as a Kiosk / required app so users can't uninstall it.

After deploy, verify on a device: app launches, hardware scanner trigger
feeds keystrokes into inputs, camera scanning works, and the app keeps
functioning with wifi off mid-session (the PWA service worker in
`public/sw.js` caches the shell).

## Local build (alternative)

You need Node 18+, JDK 17, and the Android SDK (Android Studio installs it).

```bash
# from project root
npm install
npx cap sync android

# build a signed release APK using your local keystore
cd android
./gradlew :app:assembleRelease \
  -PCR_KEYSTORE_PATH=/absolute/path/to/countright.keystore \
  -PCR_KEYSTORE_PASSWORD=<PASSWORD> \
  -PCR_KEY_ALIAS=countright \
  -PCR_KEY_PASSWORD=<PASSWORD>

# output:
ls -lh app/build/outputs/apk/release/app-release.apk
```

For an unsigned debug build (handy for `adb install -r`):

```bash
cd android && ./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

## How the project is wired

- `capacitor.config.ts` — Capacitor config. `server.url` points to the
  published Lovable site, so the native shell is just a WebView pointed
  at `/m`. `allowNavigation` whitelists Supabase + Lovable subdomains.
- `capacitor-www/` — placeholder webDir required by Capacitor. We don't
  ship a static build into the APK — content is loaded remotely.
- `android/` — full native project generated by `npx cap add android`.
  Includes `gradlew`, `gradlew.bat`, `settings.gradle`, project + app
  level `build.gradle`, gradle wrapper jar, the `app/` module, and
  Capacitor's standard plumbing.
- `android/app/build.gradle` — adds a `signingConfigs.release` block
  that reads `CR_KEYSTORE_*` from `-P` properties or env vars (used by
  the CI workflow).
- `.github/workflows/android.yml` — CI: installs JS deps, runs
  `npx cap sync android`, decodes the keystore from the secret, and
  runs `./gradlew :app:assembleRelease :app:bundleRelease`.

## Troubleshooting

- **`gradlew: Permission denied`** — `chmod +x android/gradlew` (the CI
  workflow does this automatically).
- **`SDK location not found`** locally — install Android Studio or set
  `ANDROID_HOME` / `ANDROID_SDK_ROOT`. CI uses the runner's pre-installed SDK.
- **`Keystore was tampered with, or password was incorrect`** — the
  `ANDROID_KEYSTORE_PASSWORD` secret doesn't match what you used in
  `keytool -genkeypair`.
- **App opens to a blank screen** — check the device can reach
  `https://aagcountright.lovable.app/m` (no proxy/captive portal).
